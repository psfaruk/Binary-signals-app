# Deep Analysis: Why Predictions Were Inverted

**Date:** 2026-07-10
**Symptom reported by user:** "Prediction dekhay green, actually candle hoy red. Prediction dekhay red, actually candle hoy green."

---

## Root Causes (in priority order)

### 1. Theories Are Predominantly Continuation-Biased

The original `analyze_eoc.py` ran 9 theories, of which **6 were continuation-biased** (predict the next candle continues in the same direction as the last):

| Theory | Bias | Original Score |
|--------|------|----------------|
| CON    | Continuation | +3 (3 same-dir candles → continue) |
| MICRO  | Continuation | +3 (buyers won → next up too) |
| RUN    | Continuation | +3 (tick-weighted buyer pressure → continue) |
| MST    | Continuation | +1 (trend → continue) |
| GAP    | Continuation | +2 (gap + close same dir → continue) |
| SHIFT  | Mixed        | +2 / +3 |
| REV    | Reversal     | +3 / +4 (key-level rejection) |
| TRAP   | Reversal     | +3 / +4 (failed breakout) |
| LAST   | Reversal     | +2 / +3 (exhaustion) |

**Net effect:** Every strong directional pattern (3 same-direction candles, strong buyer pressure, gap up + close up) generated a CONTINUATION vote, drowning out the reversal theories.

### 2. Quotex OTC Markets Are Mean-Reverting at 60s Timeframe

OTC forex quotes on Quotex are synthetically generated by the broker with a slight mean-reverting drift. At the 60-second candle timeframe, after a directional move:
- Profit-taking pulls price back
- Broker algorithms dampen extended trends
- 3-candle streaks often signal exhaustion, not continuation

This is the **opposite** of what continuation theories assume. A 3-bullish-candle streak in OTC more often precedes a bearish reversal candle than a 4th bullish continuation candle.

### 3. The `MICRO` Theory Logic Is Backward for Mean-Reverting Markets

Original code:
```python
if buy_pct >= 65 and net > 0:
    score += 3  # CALL — next candle continues up
```
Translation: "If buyers dominated AND the candle closed up, predict the NEXT candle also closes up."

This logic is identical to CON — it just uses tick-level data instead of OHLC. In a mean-reverting market, a strongly bullish closed candle is a **reversal signal**, not a continuation signal. The original MICRO theory amplified the same wrong bias using more granular data.

### 4. No Adaptive Feedback Loop

The original code had `ZONE_LOSS_GUARD = 3` which suppressed signals to NEUTRAL after 3 consecutive wrong predictions in the same regime/zone. But this only **stopped guessing** — it did not learn that the model was systematically wrong and flip the direction.

A 100%-inverted model contains 100% useful information — every CALL is actually a PUT and vice versa. Throwing that information away (by going NEUTRAL) wastes the edge.

## Fixes Applied

### Fix 1: Adaptive Inversion (the safety net)

Added a `recent_accuracy` parameter to `analyze_eoc()`. If the model's accuracy over the last 20 graded predictions for THIS asset/period falls below 40% with at least 8 samples, the final signal is automatically flipped CALL ↔ PUT.

```python
# analyze_eoc.py — main entry point
if (recent_accuracy is not None
        and recent_n >= 8
        and recent_accuracy < 0.40):
    majority = "PUT" if majority == "CALL" else "CALL"
    net = -net
    flipped = True
    all_reasons.append(f"INVERT:+1 {majority} adaptive-flip ...")
```

The flipped signal carries `_flipped: True` so the UI / logs can show when the safety net engaged.

**Why this works:** A model that is wrong 80%+ of the time is a model that has correctly identified the direction but mislabeled it. Flipping converts a 20%-accurate model into an 80%-accurate model. The threshold (40% over 8+ samples) is conservative enough that random noise (50% accuracy) won't trigger it.

### Fix 2: Strengthened Reversal Theories

| Theory | Old Score | New Score | Rationale |
|--------|-----------|-----------|-----------|
| REV (basic)       | +3 / -3 | +4 / -4 | Reversal is the edge in mean-reverting markets |
| REV (at key level)| +4 / -4 | +5 / -5 | Double-bottom/top at swing level is highest-conviction |
| TRAP (heavy)      | +4 / -4 | +5 / -5 | Failed breakout with heavy reversal is highest-conviction |
| TRAP (basic)      | +3 / -3 | +4 / -4 | Same |
| LAST (exhaustion) | +3 / -3 | +4 / -4 | Final-tick exhaustion reversal is high-conviction |
| LAST (overextended)| +2 / -2 | +3 / -3 | Same |

### Fix 3: Weakened CON Theory

Reduced CON's continuation score from `+3` to `+2`. The continuation logic itself is sound, but in this market it is wrong more often than right — so it should not be the dominant vote. Exhaustion sub-cases of CON kept their original +/-1 weights.

### Fix 4: New MEAN Theory (mean-reversion detector)

Added a brand-new `_theory_mean()` that explicitly votes for REVERSAL when:
1. **Oversized body** (body > 1.4× ATR) — exhaustion candidate, vote reversal (+3)
2. **RSI extreme** (≥75 or ≤25) — overbought/oversold, vote reversal (+2)
3. **Failed-continuation wick** (long wick opposite to body direction) — buyers/sellers failed, vote reversal (+2)
4. **4+ same-direction candles** — trend exhaustion, vote reversal (+2)

This is the symmetric opposite of CON, and in a mean-reverting market it should win more often than CON.

### Fix 5: db.recent_accuracy() helper

Added a lightweight query to `db.py` that returns `(accuracy_float, sample_count)` for the last N graded CALL/PUT signals for an asset/period. Draws are excluded from the denominator. Used by both `feed.py` and `sim_feed.py` to feed the adaptive-inversion logic.

### Fix 6: feed.py + sim_feed.py wired up

Both feeds now call `_db.recent_accuracy(asset, period, n=20)` before every EOC analysis and pass the result into `analyze_eoc(..., recent_accuracy=acc, recent_n=n_acc)`. Wrapped in try/except so a DB issue never blocks signal generation.

## Expected Behavior After Update

1. **First 8 candles after start:** Same as before — no flip possible (insufficient samples). The strengthened REV/TRAP/LAST and new MEAN theory should already improve accuracy.
2. **After 8 candles:** If accuracy is naturally ≥40%, no flip — the model is working.
3. **If accuracy is < 40%:** Adaptive inversion kicks in. Every CALL becomes PUT and vice versa. A 20%-accurate model becomes an 80%-accurate model.
4. **The flip is dynamic:** As the market regime changes and the model's accuracy recovers, the flip automatically disengages (accuracy rises above 40%).
5. **Per-asset/period:** Each pair has its own accuracy tracking, so a pair where the model works is never flipped because of a different pair's failure.

## What to Watch

- **`_flipped: True`** flag in the prediction dict — visible in the reasons log as `INVERT:+1 ... adaptive-flip (recent_acc=...)`
- **signal_log table** — accuracy column should improve from current levels
- **theory_perf** — over time, MEAN and REV/TRAP should rise in accuracy rank vs CON

## What Could Still Go Wrong

- If the market genuinely trends for an extended period, MEAN theory will keep voting reversal and lose. The adaptive-inversion safety net should catch this (flip back when accuracy drops), but watch the first 8 candles of any new trend.
- The 40% threshold is a guess based on the user's "100% inverted" report. If real accuracy hovers around 40-45%, the flip will toggle on/off frequently. Tune the threshold in `analyze_eoc.py` if needed.
- The `recent_accuracy` query hits the DB on every EOC. With ~38 always-on streams closing simultaneously each minute, this is ~38 queries/minute — should be fine for SQLite but worth watching.
